> [!NOTE]
> **Note:** You can get higher rate limits by subscribing to [Google One](https://one.google.com/intl/en_us/about). Or, you can [add a Gemini API key](https://developer.android.com/studio/gemini/add-api-key) to expand your context window and pay per token.

![](https://developer.android.com/static/studio/gemini/images/gemini-onboarding.png)

To get started with Gemini in Android Studio, follow these steps:

1. [Download Android Studio](https://developer.android.com/studio). Note that some features are under active development and only available through [Studio Labs](https://developer.android.com/studio/gemini/labs) or a [preview
   version of Android Studio](https://developer.android.com/studio/preview). See [Features](https://developer.android.com/studio/gemini/features) to check what version of Android Studio you need for the features you're interested in using.
2. To launch Gemini, open or start an Android Studio project and click **Agent** ![](https://developer.android.com/static/studio/images/agent-icon.png). Alternatively, click **View \> Tool Windows \> Agent**.
3. Sign in to your Google Account when asked, if you aren't already signed in.
4. Complete the onboarding flow.
   1. Select **Gemini for individuals** when given the choice of product tier.
   2. For the best results, select **Use all Gemini features** to allow project context sharing. By default, this setting applies to all projects that you use Gemini for. To update the setting or request to configure context sharing on a per project basis, go to the [Gemini
      settings](https://developer.android.com/studio/gemini/configure-gemini).
